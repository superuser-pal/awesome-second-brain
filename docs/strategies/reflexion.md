# Reflexion

## Strategy

Answer concisely, critique your reasoning briefly, and provide a refined answer.

## When to apply

- Drafting decisions, plans, or written work where a self-review pass adds value.
- Key decision records where the first-pass answer benefits from explicit critique.
- Situations where the user wants to see potential weaknesses surfaced before committing.
- Review writing (peer reviews, self-assessments) where balanced critique is expected.

## Trade-offs

- **Pro**: Built-in self-correction catches obvious flaws before the user does.
- **Pro**: Transparent — the critique is visible and auditable.
- **Con**: Adds a full critique+revise cycle to every response — overhead for simple tasks.
- **Con**: The critique can be superficial if the original answer was already well-reasoned.
