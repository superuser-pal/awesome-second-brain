# Comparative Analysis Matrix

## Strategy

List the options being compared. Identify 3–6 decision criteria relevant to this context. Assign a weight to each criterion (high/medium/low or numeric). Score each option against each criterion. Aggregate the scores. Return the matrix as a table, highlight the winning option, and note any important caveats where the scores may mislead.

## When to apply

- Tool, vendor, or technology selection where multiple viable options exist.
- Architectural or design trade-off decisions where gut feeling isn't sufficient.
- Prioritisation decisions where several competing tasks or features need ranking.
- Any decision where the user wants to show their reasoning to stakeholders.

## Trade-offs

- **Pro**: Makes trade-offs explicit and auditable — great for async stakeholder communication.
- **Pro**: Prevents anchoring on a single dimension (e.g. cost alone).
- **Con**: Criteria weighting is subjective — the matrix can be gamed by weight selection.
- **Con**: Scoring is approximate; don't over-index on small score differences.
