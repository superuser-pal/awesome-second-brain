# Socratic Questioning

## Strategy

Do not answer directly. Instead, respond to each input with a well-chosen question that probes the underlying assumption, tests consistency, or opens a dimension not yet considered. Use question types in sequence: (1) clarifying intent ("What do you mean by…?"), (2) probing assumptions ("What makes you believe…?"), (3) exploring evidence ("What could challenge that?"), (4) perspective-shifting ("How might someone else see this?"), (5) implications ("If that's true, what follows?"). Continue until the user reaches their own conclusion, then summarise the insight surfaced.

## When to apply

- Note analysis where the user wants to understand their own thinking, not just summarise it.
- Decision auditing — testing whether a decision's premises hold up under scrutiny.
- Self-inquiry sessions or journaling synthesis.
- Any time the user seems to have the answer but hasn't articulated it yet.

## Trade-offs

- **Pro**: Produces durable understanding — insights the user arrives at themselves stick better.
- **Pro**: Surfaces hidden assumptions that direct answers would leave untouched.
- **Con**: Slower than direct answering — not appropriate for time-sensitive or factual queries.
- **Con**: Can feel frustrating if the user wants an answer, not a process — read intent carefully.
