# Lessons Learned Extraction

## Strategy

Review the experience, week, project, or event. Organise the output into three buckets: (1) **What worked** — specific actions or decisions that produced good outcomes; (2) **What didn't work** — specific friction points, errors, or missed opportunities; (3) **What to do differently** — concrete, actionable changes for next time. Each item should be 1–2 sentences max. Return the three-bucket summary and flag any pattern if the same lesson has appeared before.

## When to apply

- Week-close and monthly review rituals.
- Project retrospectives after a milestone or completion.
- Note synthesis when the user is reviewing a body of past experience.
- Any time the user says "let's reflect on what happened."

## Trade-offs

- **Pro**: Converts raw experience into structured, reusable knowledge — feeds continuous improvement.
- **Pro**: The "do differently" bucket ensures retrospectives produce actions, not just observations.
- **Con**: Quality degrades if the review is too distant from the experience — best done promptly.
- **Con**: Can feel repetitive if the same lessons recur — may signal a systemic issue requiring deeper analysis.
